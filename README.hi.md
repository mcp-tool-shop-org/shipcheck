## क्यों

"हो गया" का मतलब पहले यह होता था कि कोड काम करता है। अब, यह पर्याप्त नहीं है। एक उत्पाद कोड + सुरक्षा + त्रुटि प्रबंधन + दस्तावेज़ + पहचान + शिपिंग स्वच्छता है। शिपचेक मानक निर्धारित करता है।

## इसमें क्या है

| मानक | यह क्या कवर करता है |
|----------|----------------|
| [Ship Gate](templates/SHIP_GATE.md) | 31 हार्ड-गेट + 4 सॉफ्ट-गेट प्री-रिलीज़ चेकलिस्ट |
| [Error Contract](contracts/error-contract.md) | 2-स्तरीय संरचित त्रुटि मानक जिसमें कोड रजिस्ट्री शामिल है |
| [Security Baseline](templates/SECURITY.md) | रिपोर्ट ईमेल, प्रतिक्रिया समयरेखा, खतरे का दायरा |
| [Handbook](templates/HANDBOOK.md) | जटिल उपकरणों के लिए परिचालन क्षेत्र मैनुअल |
| [Scorecard](templates/SCORECARD.md) | पूर्व/पश्चात सुधार स्कोरिंग |
| [Adoption Guide](ADOPTION.md) | किसी भी रिपॉजिटरी पर <30 मिनट में शिपचेक लागू करें |

## सीएलआई उपयोग

```bash
npx @mcptoolshop/shipcheck init        # Copy templates into current repo
npx @mcptoolshop/shipcheck audit       # Check SHIP_GATE.md progress
npx @mcptoolshop/shipcheck dogfood     # Check dogfood freshness (Gate F)
npx @mcptoolshop/shipcheck front-door  # Verify the AI-native front door (Gate G)
npx @mcptoolshop/shipcheck help        # Show help
npx @mcptoolshop/shipcheck --version   # Show version
```

रंगीन पाठ के बजाय संरचित JSON त्रुटि आउटपुट प्राप्त करने के लिए `SHIPCHECK_JSON=1` सेट करें।

## त्वरित शुरुआत

1. [अपनाना.एमडी](ADOPTION.md) पढ़ें
2. अपने रिपॉजिटरी रूट में `npx @mcptoolshop/shipcheck init` चलाएं
3. `SHIP_GATE.md` में लागू वस्तुओं को चिह्नित करें, गैर-लागू वस्तुओं को `SKIP:` से चिह्नित करें
4. `npx @mcptoolshop/shipcheck audit` चलाएं - जब सभी हार्ड गेट पास हो जाएं तो 0 पर समाप्त होता है
5. ऑडिट के पास होने पर शिप करें

## यह कैसे काम करता है

**हार्ड गेट** (ए-डी) रिलीज़ को रोकते हैं:

- **ए. सुरक्षा आधारभूत संरचना** - SECURITY.md, खतरे का मॉडल, कोई गुप्त जानकारी नहीं, कोई टेलीमेट्री नहीं, डिफ़ॉल्ट सुरक्षा स्थिति
- **बी. त्रुटि प्रबंधन** - संरचित त्रुटि आकार (कोड/संदेश/संकेत/पुन: प्रयास करने योग्य), सुरक्षित आउटपुट, सहज गिरावट
- **सी. ऑपरेटर दस्तावेज़** - README, CHANGELOG, LICENSE, उपकरण प्रलेखन
- **डी. शिपिंग स्वच्छता** - सत्यापन स्क्रिप्ट, संस्करण संरेखण, निर्भरता स्कैनिंग, लॉकफ़ाइल

**सॉफ्ट गेट** (ई) रिलीज़ को नहीं रोकता है लेकिन "संपूर्ण" को परिभाषित करता है:

- **ई. पहचान** - लोगो, अनुवाद, लैंडिंग पृष्ठ, रिपॉजिटरी मेटाडेटा

**गेट एफ - डॉगफूड ताजगी** (वैकल्पिक, dogfood-labs की आवश्यकता है):

- एक ताज़ा, सत्यापित, पासिंग डॉगफूड रिकॉर्ड के लिए जाँच करता है
- प्रवर्तन मोड का समर्थन करता है: `आवश्यक`, `केवल चेतावनी`, `छूट`
- कॉन्फ़िगर करने योग्य ताजगी विंडो (डिफ़ॉल्ट: 30 दिन)

**गेट जी - एआई-नेटिव फ्रंट डोर** (वैकल्पिक, `@mcptoolshop/site-theme` >=2.0.0 की आवश्यकता है):

- सत्यापित करता है कि रिपॉजिटरी का एआई-नेटिव फ्रंट डोर (README / AGENTS.md / llms.txt) सच बताता है - ऑपरेटर दस्तावेज़ (सी) और पहचान (ई) गेट के लिए मशीन-पठनीय पूरक
- साइट-थीम के [`फ्रंट-डोर`](https://github.com/mcp-tool-shop-org/site-theme) सत्यापनकर्ता (`verify({ root })`) पर प्रत्यायोजित करता है, जो प्रलेखित दावों को साक्ष्य चैनलों तक पहुंचाता है और जोखिम-क्रमित स्कोरकार्ड लौटाता है
- गंभीरता के अनुसार गणनाओं को सतह प्रदान करता है (विरोधाभासी · अप्रमाणित · पुराना · अतिरेक · स्वच्छता · शैली) प्लस गेट निर्णय; **विरोधाभासी / अप्रमाणित / पुराने दावों पर विफल रहता है (निकास 1)**
- साइट-थीम एक **वैकल्पिक सहकर्मी निर्भरता** है (एक हार्ड निर्भरता इस शून्य-निर्भर सीएलआई में एस्ट्रो को खींच लेगी)। जब यह स्थापित नहीं होता है, तो गेट **अनुग्रहपूर्वक छोड़ देता है** (निकास 0) - यह कभी भी ऑडिट को क्रैश नहीं करता है

गेट बताता है कि **क्या** सत्य होना चाहिए, न कि **कैसे** इसे लागू किया जाए। प्रयोज्यता टैग (`[सभी]`, `[npm]`, `[mcp]`, `[cli]`, `[डेस्कटॉप]`, `[vsix]`, `[कंटेनर]`) उन रिपॉजिटरी पर चेकबॉक्स शर्मिंदगी को रोकते हैं जहां आइटम लागू नहीं होते हैं।

## त्रुटि अनुबंध एक नज़र में

**स्तर 1 - आकार (हर जगह अनिवार्य):**

```json
{
  "code": "INPUT_TEXT_EMPTY",
  "message": "Text must not be empty",
  "hint": "Provide at least one character of text",
  "retryable": false
}
```

**स्तर 2 - आधार प्रकार + निकास कोड (सीएलआई/एमसीपी/डेस्कटॉप):**

| निकास कोड | अर्थ |
|-----------|---------|
| 0 | OK |
| 1 | उपयोगकर्ता त्रुटि (खराब इनपुट, कॉन्फ़िगरेशन गायब) |
| 2 | रनटाइम त्रुटि (क्रैश, बैकएंड विफलता) |
| 3 | आंशिक सफलता (कुछ आइटम सफल हुए) |

त्रुटि कोड नामस्थानित उपसर्गों का उपयोग करते हैं: `IO_`, `CONFIG_`, `PERM_`, `DEP_`, `RUNTIME_`, `PARTIAL_`, `INPUT_`, `STATE_`. कोड जारी होने के बाद स्थिर होते हैं।

## विश्वास मॉडल

**स्पर्श किया गया डेटा:** वर्तमान कार्यशील निर्देशिका में `package.json`, `pyproject.toml` और `SHIP_GATE.md` पढ़ता है। केवल वर्तमान निर्देशिका में टेम्पलेट फ़ाइलें (`SHIP_GATE.md`, `SECURITY.md`, `CHANGELOG.md`, `SCORECARD.md`) लिखता है।
**कोई नेटवर्क अनुरोध नहीं।** सभी संचालन स्थानीय फ़ाइल रीड और राइट हैं।
**कोई गुप्त जानकारी प्रबंधन नहीं।** क्रेडेंशियल को नहीं पढ़ता, संग्रहीत या प्रसारित करता है।
**कोई टेलीमेट्री** एकत्र या भेजी नहीं जाती है।
